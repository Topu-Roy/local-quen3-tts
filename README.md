# Qwen3-TTS TUI

A **Textual** terminal user interface for **Qwen3-TTS** voice cloning on CPU. Clone any voice from a 3-second audio sample and synthesize speech across 10 languages.

![CPU only](https://img.shields.io/badge/CPU-only-orange)
![Python](https://img.shields.io/badge/Python-3.9+-blue)
![License](https://img.shields.io/badge/License-Apache%202.0-green)

---

## Features

- **Voice cloning** — Provide a `.wav` reference + `.txt` transcript, and the model reproduces that voice
- **10 languages** — Chinese, English, Japanese, Korean, German, French, Russian, Portuguese, Spanish, Italian
- **Auto language detection** — Just type, the model figures out the language
- **Session history** — Every generation logged to `logs/YYYY-MM-DD.jsonl`
- **Multi-line input** — Shift+Enter for newlines, Enter to generate
- **Two model sizes** — `0.6B` (faster) or `1.7B` (better quality) — set in `config.toml`
- **Auto-format conversion** — Drop `.webm`, `.mp3`, `.ogg`, etc. into `samples/`; bootstrap auto-converts to `.wav`

- **Python 3.9+**

## Setup

```bash
# 1. Clone
git clone https://github.com/Topu-Roy/local-quen3-tts.git
cd local-quen3-tts

# 2. Bootstrap (creates venv, installs deps, downloads models)
python bootstrap.py

# 3. Activate venv (printed by bootstrap)
# Windows:
.\venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate

# 4. Re-run bootstrap (continues from Phase 2)
python bootstrap.py
```

Bootstrap runs through 7 phases:

| Phase | What it does |
|---|---|
| 1 | Checks Python version, creates venv, loads config |
| 2 | Detects CPU cores and RAM |
| 3 | Installs PyTorch CPU, clones Qwen3-TTS from GitHub, installs deps |
| 4 | Checks for ffmpeg |
| 5 | Downloads tokenizer + base model from HuggingFace (~1-2 GB) |
| 6 | Validates voice samples in `samples/`, auto-converts non-wav formats |
| 7 | Writes `bootstrap.lock` and prints launch command |

## Voice Samples

Place audio samples in the `samples/` folder. Each sample needs a matching transcript:

```
samples/
├── my-voice.wav       # audio (16kHz mono preferred, 3-30s)
└── my-voice.txt       # transcript of what's spoken in the audio
```

Supported input formats: `.wav`, `.webm`, `.mp3`, `.ogg`, `.flac`, `.m4a`, `.opus` — non-wav files are auto-converted by bootstrap.

---

## Configuration

Edit `config.toml` (generated from `config.toml.default` on first bootstrap run):

```toml
[model]
size = "0.6b"           # "0.6b" or "1.7b"
device = "cpu"

[voice]
default = "samples/"    # first valid .wav in samples/, or explicit path

[output]
format = "wav"          # "wav" or "mp3" (mp3 requires ffmpeg)
naming = "timestamp"    # "timestamp" or "prompt-prefix"
folder = "output"
auto_play = true        # play generated audio immediately

[session]
log_history = true      # save to logs/YYYY-MM-DD.jsonl
show_history = true     # show panel in TUI
```

## Usage

```bash
python tts.py
```

### Keybindings

| Key | Action |
|---|---|
| `Enter` | Generate speech from typed text |
| `Shift+Enter` | Insert newline in input |
| `F2` | Select voice from samples |
| `F3` | Open output folder in file manager |
| `F4` | Play last generated audio |
| `Esc` | Cancel current generation |
| `Ctrl+Q` | Quit |

### Output naming

- **Timestamp mode:** `output/2026-05-25_14-32-01.wav`
- **Prompt-prefix mode:** `output/hello-how-are-you_14-32-01.wav`

### Session logs

Each generation is logged to `logs/YYYY-MM-DD.jsonl`:

```json
{"time": "14:32:01", "text": "Hello world", "file": "output/2026-05-25_14-32-01.wav", "voice": "my-voice.wav", "device": "cpu", "duration_ms": 3241}
```

## Project Structure

```
local-quen3-tts/
├── bootstrap.py          # One-time setup (venv, deps, models, samples)
├── tts.py                # Main app (TUI + ModelEngine + SessionLog + AudioPlayer)
├── config.toml.default   # Default config template
├── requirements.txt      # Python dependencies
├── APP.md                # Full implementation plan
├── VERSION               # Semantic version
├── .gitignore
├── samples/              # Voice samples (.wav + .txt per voice)
├── logs/                 # Session history (.jsonl)
└── output/               # Generated audio (.wav)
```

Generated by bootstrap (not committed):

```
├── venv/                 # Python virtual environment
├── models/
│   ├── tokenizer/        # Qwen3-TTS-Tokenizer-12Hz
│   └── 0.6b-base/        # Qwen3-TTS-12Hz-0.6B-Base (or 1.7b-base)
├── Qwen3-TTS/            # Cloned GitHub source
├── config.toml           # User config (generated from .default)
└── bootstrap.lock        # Bootstrap completion marker
```

## Technical Stack

| Component | Technology |
|---|---|
| Inference | [Qwen3-TTS](https://github.com/QwenLM/Qwen3-TTS) (via `qwen_tts` package) |
| TUI | [Textual](https://textual.textualize.io/) |
| Audio playback | `winsound` (Windows) / `afplay` (macOS) / `aplay` (Linux) |
| Model download | HuggingFace Hub |
| Audio processing | ffmpeg, scipy, soundfile |

---

## Out of Scope

- No audio recording (provide pre-recorded samples)
- No batch mode
- No web UI or API server
- No model switching mid-session
- No waveform visualization
- No builtin/default voice (all voices require a reference sample)
- No CUDA/GPU support
